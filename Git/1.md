## 🔧 环境准备 Setup

### 1. 安装 Git
- 下载地址：https://git-scm.com/download/win
- 安装完后验证：
```bash
git --version
# 显示 git version 2.x.x 即成功
```

### 2. 配置身份 Identity
第一次使用需要告诉 Git 你是谁：
```bash
git config --global user.email "你的邮箱"
git config --global user.name "你的名字"
```

---

## 🚀 第一次连接 GitHub（只需做一次）

本地文件夹如果是从 GitHub 下载 zip 解压的（不是 clone 的），需要手动连接：

```bash
# 1. 初始化 Git
git init

# 2. 连接远程 GitHub repo
git remote add origin https://github.com/你的用户名/你的repo名.git

# 3. 拉取 GitHub 上的内容
git fetch origin

# 4. 强制同步（本地覆盖为 GitHub 版本）
git reset --hard origin/main

# 5. 统一分支名为 main
git branch -M main
```

## 📝 日常记笔记流程（每次使用）

### 第一步：打开项目
VS Code → `File → Open Folder` → 选你的本地项目文件夹

### 第二步：写笔记
在 VS Code 里新建或编辑 `.md` 文件

### 第三步：Push 到 GitHub
打开终端（`Ctrl + ` ` `）运行：

```bash
git add .
git commit -m "描述你做了什么，比如：Add Delta Lake notes"
git push origin main
```

### ✅ 常见返回结果
| 返回信息 | 意思 |
|---------|------|
| `main -> main` | ✅ 成功 push 到 GitHub |
| `Everything up-to-date` | ✅ 本地和 GitHub 已同步，没有新变化 |
| `fatal: not a git repository` | ❌ 没有初始化 Git，需要运行 `git init` |

---

## ⚠️ 注意事项

### 删文件前要小心！
`git add .` 会把**所有变化**都记录，包括删除！

所以 push 之前养成习惯先运行：
```bash
git status
```
确认没有意外删除的文件再 push。

### 看到 `deleted: 文件名` 要停下来确认 ⚠️
